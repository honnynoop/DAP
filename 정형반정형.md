# 데이터 유형별 페이스북 예시

## 1. 정형 데이터 (Structured Data)

### 특징
- 고정된 스키마, 행과 열로 구성
- RDBMS에 저장
- SQL로 쉽게 조회/분석 가능

### 페이스북 예시: 사용자 정보 테이블

**데이터 형식: RDB (MySQL/PostgreSQL)**
```sql
-- users 테이블
user_id | name        | email                | join_date  | age | gender
--------|-------------|----------------------|------------|-----|-------
1001    | 홍길동      | hong@email.com       | 2020-01-15 | 28  | M
1002    | 김영희      | kim@email.com        | 2019-05-20 | 32  | F
1003    | 이철수      | lee@email.com        | 2021-03-10 | 25  | M

-- friends 테이블 (친구 관계)
user_id | friend_id | friendship_date
--------|-----------|----------------
1001    | 1002      | 2020-02-01
1001    | 1003      | 2020-03-15
```

## 2. 반정형 데이터 (Semi-Structured Data)

- 유연한 스키마 (속성 추가/삭제 가능)

## 3. 비정형 데이터 (Unstructured Data)

### 특징
- 정해진 구조/형식 없음
- 메타데이터 없음
- 전체 데이터의 80% 이상 차지

### 페이스북 예시

**데이터 형식별 구체적 예시:**

#### 이미지/사진
```
데이터 형식: JPEG, PNG, GIF, WEBP
파일: profile_1001.jpg (사용자 프로필 사진)
```

#### 음성

## 📊 비교 요약표

| 구분 | 정형 | 반정형 | 비정형 |
|------|------|--------|--------|
| **구조** | 고정 스키마 | 유연한 스키마 | 구조 없음 |
| **저장소** | RDBMS | NoSQL, 파일시스템 | Object Storage, HDFS |
| **페이스북 예** | 사용자 프로필, 친구 관계 | 게시물, 댓글, API 로그 | 사진, 동영상, 텍스트 |
| **형식** | SQL Table | JSON, XML, CSV | JPEG, MP4, TXT |
| **분석 난이도** | 쉬움 (SQL) | 중간 (파싱 필요) | 어려움 (전처리 필수) |
| **비율** | ~5% | ~15% | ~80% |

## 1. 정형 데이터 (Structured Data)

### 특징
- 고정된 스키마, 행과 열로 구성
- RDBMS에 저장 (Oracle, MySQL, PostgreSQL)
- SQL로 쉽게 조회/분석 가능
- 데이터 타입 명확 (숫자, 문자, 날짜 등)

## 📊 비교 요약표

| 구분 | 정형 | 반정형 | 비정형 |
|------|------|--------|--------|
| **구조** | 고정 스키마 | 유연한 스키마 | 구조 없음 |
| **메타데이터** | 스키마 정의 | 자체 포함 (태그) | 없음 |
| **저장소** | RDBMS | NoSQL, 파일시스템 | Object Storage, HDFS |
| **페이스북 예** | 사용자 프로필, 친구관계 | 게시물, 댓글, 로그 | 사진, 동영상, 채팅 |
| **센서 예** | - | IoT JSON, CSV 로그 | CCTV 영상, 음향 |
| **형식** | SQL Table | JSON, XML, CSV | JPEG, MP4, TXT, WAV |
| **분석 도구** | SQL | JSON Parser, Spark | NLP, CV, DL |
| **분석 난이도** | ⭐ 쉬움 | ⭐⭐ 중간 | ⭐⭐⭐ 어려움 |
| **전체 비율** | ~5% | ~15% | ~80% |
| **쿼리 언어** | SQL | NoSQL, JSONPath | - |

## 💡 실무 처리 예시

### 정형 데이터 처리
```sql
-- SQL로 직접 분석
SELECT age, gender, COUNT(*) as user_count
FROM users 
WHERE join_date >= '2020-01-01'
GROUP BY age, gender
ORDER BY user_count DESC;
```

### 반정형 데이터 처리

```python
hashtags = [tag for post in posts for tag in post['hashtags']]

# 센서 데이터 (CSV)
df = pd.read_csv('sensor_data.csv')
df['timestamp'] = pd.to_datetime(df['timestamp'])
avg_temp = df.groupby('sensor_id')['temperature'].mean()
```

## 📁 데이터 형식별 정리

### 정형
- `.sql`, `.db`, `.mdb` (데이터베이스 파일)

### 반정형
- `.json` - JavaScript Object Notation
- `.xml` - eXtensible Markup Language
- `.csv` - Comma-Separated Values
- `.yaml` - YAML Ain't Markup Language
- `.parquet` - Apache Parquet (컬럼 기반)
- `.avro` - Apache Avro (Row 기반)

### 비정형
- **이미지**: `.jpg`, `.png`, `.gif`, `.webp`, `.tiff`
- **동영상**: `.mp4`, `.avi`, `.mov`, `.mkv`, `.webm`
- **오디오**: `.mp3`, `.wav`, `.flac`, `.aac`, `.ogg`
- **문서**: `.pdf`, `.doc`, `.txt`, `.hwp`
- **기타**: `.dat`, `.bin` (바이너리)
